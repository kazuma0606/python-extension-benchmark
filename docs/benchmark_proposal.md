# Python拡張ライブラリ 多言語実装ベンチマーク 企画書
*Monorepo構成・再現可能な環境・定量的比較*

---

## 1. 背景と目的

Pythonは機械学習・データ処理・Web開発など広範な領域で利用されているが、その実行速度はインタープリタ言語の特性上、ボトルネックになりやすい。この課題を解決するため、C・C++・Cython・Rustといった低レベル言語でPython拡張ライブラリを実装する手法が実用されている。

本企画は、同一処理を各言語実装で実行し定量的にパフォーマンスを比較することで、以下を明らかにすることを目的とする。

- 各言語実装の処理速度・メモリ効率の差異
- 処理特性（CPUバウンド / メモリバウンド / 並列処理）によるパフォーマンス傾向
- PythonのGIL問題が並列処理に与える実測的な影響
- 実装コスト（学習コスト・コード量）とパフォーマンス向上のトレードオフ

---

## 2. 対象技術スタック

Pure Pythonを基準値（ベースライン）として、各実装の相対パフォーマンスを計測する。

| 言語 / ツール | Pythonバインディング | メモリ安全性 | 学習コスト | 主な用途 |
|---|---|---|---|---|
| Pure Python | —（本体） | 高（GC管理） | 低 | 基準値計測 |
| C（ctypes） | ctypes / CFFI | 低 | 高 | ローレベル制御 |
| C++（pybind11） | pybind11 | 低〜中 | 高 | 既存C++資産活用 |
| Cython | Cython（変換） | 中 | 中 | 段階的移行 |
| Rust（PyO3） | PyO3 | 最高 | 中〜高 | 安全×高速 |
| Fortran | f2py（numpy） | 中 | 高 | 科学数値計算 |

---

## 3. ベンチマークシナリオ

処理特性が異なる3つのシナリオを設定し、各言語の得意・不得意領域を多角的に評価する。

| # | シナリオ名 | 処理内容 | 測定ポイント |
|---|---|---|---|
| 1 | 数値計算 | 素数探索（エラトステネスの篩）・行列積 | 純粋なCPU演算速度差 |
| 2 | メモリ操作 | 1000万件配列のソート・フィルタ | メモリ管理効率・アロケーションコスト |
| 3 | 並列処理 | マルチスレッドでの分散計算 | GIL問題・スレッド並列性の差 |

### シナリオ選定の理由

- **シナリオ1（数値計算）** — 純粋なCPU演算速度差が最も顕著に出やすく、各言語の基礎性能比較に最適
- **シナリオ2（メモリ操作）** — メモリ管理戦略の違いが如実に表れ、RustのゼロコストアブストラクションとCのロースタックが有利な領域
- **シナリオ3（並列処理）** — PythonのGIL（Global Interpreter Lock）問題という構造的な制約が可視化され、最も学びが多いシナリオ

## 4. モノレポ構成

各実装を単一リポジトリで管理し、統合ランナーから一括実行・比較できる構成とする。

```
benchmark/
├ python/         # Pure Python実装（基準値）
├ c_ext/          # C言語拡張実装
├ cpp_ext/        # C++（pybind11）実装
├ cython_ext/     # Cython実装
├ rust_ext/       # Rust（PyO3）実装
├ runner/         # 統合ベンチマーク実行スクリプト（Python）
├ results/        # 計測結果出力（JSON / CSV / グラフ）
└ docker/         # 再現性担保のためのDocker環境
```

### 設計方針

- 各言語実装は同一のインターフェース（Python関数）として呼び出し可能にする
- Docker環境で実行することで、OS・CPU・メモリ環境を統一し再現性を担保する
- 計測結果はJSON/CSVで出力し、matplotlibまたはPlotlyでグラフ生成する
- CI（GitHub Actions）で自動ベンチマーク実行・結果コミットのパイプラインを整備する

---

## 5. 計測方針

### 計測手法

- 各シナリオを100回実行し、最小値・中央値・平均値・標準偏差を記録
- ウォームアップ実行（10回）を前置し、JITやキャッシュの影響を排除
- メモリ使用量はtracemalloc（Python）およびvalgrind（C/C++）で計測
- 並列処理シナリオはスレッド数を1・2・4・8・16と変化させて計測

### 評価指標

- **実行時間（ms）** — Pure Pythonを1.0とした相対スコアで表示
- **メモリ使用量（MB）** — ピーク使用量と平均使用量
- **スループット（ops/sec）** — 単位時間あたりの処理件数
- **実装コスト** — コード行数・ビルド設定の複雑さ（定性評価）
