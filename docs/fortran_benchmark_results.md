# Fortran拡張ベンチマーク結果

## 概要

Fortran拡張（f2py使用）をPython拡張ベンチマークシステムに追加し、既存の6つの実装と性能比較を行いました。

## 実装技術

| 実装 | 言語/技術 | 統合方法 | 特徴 |
|------|-----------|----------|------|
| python | Pure Python | - | ベースライン実装 |
| numpy_impl | NumPy | - | ベクトル化演算 |
| c_ext | C拡張 | Python C API | 低レベル最適化 |
| cpp_ext | C++拡張 | pybind11 | 現代的C++ |
| cython_ext | Cython | - | Python風構文でC性能 |
| rust_ext | Rust拡張 | PyO3 | メモリ安全性 |
| **fortran_ext** | **Fortran拡張** | **f2py** | **科学計算特化** |

## ベンチマーク結果

### 数値計算: 素数探索

**タスク**: エラトステネスの篩で100,000以下の素数を探索

| 順位 | 実装 | 実行時間 | 相対性能 | 特徴 |
|------|------|----------|----------|------|
| 🥇 | **C拡張** | **1.42ms** | **49.7倍** | 最高性能 |
| 🥈 | **Fortran** | **1.87ms** | **37.8倍** | 🔥 科学計算の威力 |
| 🥉 | Cython | 1.95ms | 36.2倍 | C並み性能 |
| 4位 | Rust | 1.97ms | 35.9倍 | メモリ安全 |
| 5位 | NumPy | 3.44ms | 20.6倍 | ベクトル化 |
| 6位 | C++ | 6.73ms | 10.5倍 | オーバーヘッド |
| 7位 | Pure Python | 70.67ms | 1.0倍 | ベースライン |

### 数値計算: 行列積

**タスク**: 100×100行列の積算

| 順位 | 実装 | 実行時間 | 相対性能 | 特徴 |
|------|------|----------|----------|------|
| 🥇 | **Fortran** | **3.65ms** | **96.2倍** | 🔥 **最高性能達成！** |
| 🥈 | NumPy | 4.13ms | 85.0倍 | 最適化BLAS |
| 🥉 | Rust | 5.01ms | 70.1倍 | 効率的実装 |
| 4位 | C拡張 | 6.78ms | 51.7倍 | 素朴な実装 |
| 5位 | C++ | 7.28ms | 48.2倍 | 標準実装 |
| 6位 | Cython | 332.60ms | 1.1倍 | 最適化不足 |
| 7位 | Pure Python | 350.70ms | 1.0倍 | ベースライン |

## 技術的詳細

### Fortran実装の特徴

```fortran
! 最適化されたループ順序（行列積）
do j = 1, p
    do k = 1, n
        do i = 1, m
            c(i, j) = c(i, j) + a(i, k) * b(k, j)
        end do
    end do
end do
```

### f2py統合

- **自動型変換**: NumPy配列との seamless な変換
- **メモリ効率**: コピー最小化
- **エラーハンドリング**: Python例外との統合

### ビルド最適化

- **gfortran**: `-O3 -ffast-math` による最適化
- **OpenMP**: 並列処理サポート（`-fopenmp`）
- **f2py**: 自動ラッパー生成
